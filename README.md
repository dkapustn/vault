# 💰 Vault

Личный финансовый трекер — учёт доходов, расходов, переводов, целей, регулярных платежей, долгов и «секретной копилки». Работает полностью в браузере, данные хранятся локально на устройстве (localStorage + резервная копия в IndexedDB). Устанавливается как PWA и работает офлайн.

## Структура проекта

| Файл | Назначение |
|------|-----------|
| `index.html` | Разметка всех экранов и модальных окон |
| `styles.css` | Все стили (тёмная тема) |
| `app.js` | Вся логика приложения + регистрация service worker |
| `sw.js` | Service worker: кэширует приложение для офлайн-работы |

## Локальный запуск

Просто открой `index.html` в браузере. Для проверки PWA/офлайн-режима нужен http-сервер (service worker не работает по `file://`):

```bash
# любой статический сервер, например:
python -m http.server 8000
# затем открой http://localhost:8000
```

## Публикация на GitHub Pages

Репозиторий уже содержит статический сайт в корне, поэтому достаточно один раз включить Pages:

1. Открой репозиторий на GitHub → **Settings** → **Pages**.
2. В разделе **Build and deployment** → **Source** выбери **Deploy from a branch**.
3. **Branch**: `main`, папка `/ (root)` → **Save**.
4. Через 1–2 минуты сайт будет доступен по адресу:
   **https://kapopus.github.io/vault/**

После этого открой ссылку на телефоне и добавь на домашний экран — приложение установится и будет работать офлайн.

> При изменении файлов оболочки увеличивай версию кэша в `sw.js` (`vault-v1` → `vault-v2`), чтобы у пользователей подтянулась новая версия.

## Приватность

Все данные остаются на устройстве — никуда не отправляются. Для переноса используй **Профиль → Резервная копия** (экспорт в JSON) и **Восстановить из копии**.
